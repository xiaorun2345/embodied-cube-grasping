# LeRobot + SmolVLA + MuJoCo 启动教程

本文档用于启动当前工作区已经配置好的 LeRobot 训练框架、SmolVLA 算法环境和 MuJoCo 抓取方块 demo。

项目目录：

```bash
/home/mkls/xiao_run/lerobot_smolvla_mujoco_demo
```

Conda 环境：

```bash
/home/mkls/xiao_run/.conda-lerobot-smolvla
```

核心代码已经按职责拆开：

```text
src/lerobot_smolvla_mujoco_demo/env.py
```

只负责加载 MuJoCo XML、创建 Gymnasium 环境、执行 `reset()` / `step()` / `render()`。

```text
src/lerobot_smolvla_mujoco_demo/scripted_grasp.py
```

只负责脚本专家抓取动作，包括抓方块、移动到托盘、松开夹爪这一套规则动作。

## 1. 进入项目并激活环境

每次新开终端，先执行：

```bash
cd /home/mkls/xiao_run/lerobot_smolvla_mujoco_demo
conda activate /home/mkls/xiao_run/.conda-lerobot-smolvla
```

如果 `conda activate` 不可用，先初始化当前 shell：

```bash
source /home/mkls/anaconda3/etc/profile.d/conda.sh
conda activate /home/mkls/xiao_run/.conda-lerobot-smolvla
```

## 2. 配置 MuJoCo 离屏渲染

服务器或无显示器环境建议使用 EGL：

```bash
export MUJOCO_GL=egl
```

如果 EGL 不可用，再改用：

```bash
export MUJOCO_GL=osmesa
```

## 3. 验证 CUDA / PyTorch / SmolVLA

当前已经匹配好的版本是：

- GPU：NVIDIA GeForce RTX 4090
- Driver：535.309.01
- PyTorch：`2.7.1+cu118`
- CUDA build：`11.8`
- LeRobot：`0.5.2`

运行验证：

```bash
python - <<'PY'

import lerobot

print("torch:", torch.__version__)
print("torch cuda build:", torch.version.cuda)

if torch.cuda.is_available():
    print("gpu:", torch.cuda.get_device_name(0))
print("lerobot:", getattr(lerobot, "__version__", "unknown"))
print("smolvla:", SmolVLAConfig.__name__)
PY
```

## 4. 启动 MuJoCo 抓取方块 Demo

如果你有本地桌面、VNC 或 X11 转发，可以直接开 MuJoCo 实时窗口：

```bash
unset MUJOCO_GL
cube-grasp-viewer --loop
```

只打开场景、不播放脚本专家：

```bash
unset MUJOCO_GL
cube-grasp-viewer --manual
```

查看固定相机渲染出来的图像：

```bash
MUJOCO_GL=egl python examples/03_get_camera_image.py
```

当前 XML 里有两个相机：

```text
front
top_oblique
```

图片会保存到：

```text
outputs/camera_images/
```

实时查看固定相机视频画面，不保存图片：

```bash
unset MUJOCO_GL
python examples/04_live_camera_view.py --camera front
```

顶视角相机：

```bash
unset MUJOCO_GL
python examples/04_live_camera_view.py --camera top_oblique
```

窗口快捷键：

```text
关闭窗口 退出
```

如果你看到 `X11: Failed to open display`，说明当前终端没有图形窗口环境。需要在本地桌面、VNC、NoMachine 或 X11 转发里运行。

如果你看到 OpenCV 的 `cvNamedWindow` / `The function is not implemented`，说明当前 OpenCV 是 headless 版本。直接使用默认 MuJoCo backend，不要加 `--backend opencv`。

如果你是在纯 SSH 服务器里，没有桌面/VNC/NoMachine，但又想实时看视频画面，可以启动浏览器视频流：

```bash
MUJOCO_GL=egl python examples/05_camera_stream_server.py --camera front --host 0.0.0.0 --port 8000
```

然后在浏览器打开：

```text
http://服务器IP:8000
```

本机运行时打开：

```text
http://127.0.0.1:8000
```

键盘控制机械臂：

```bash
unset MUJOCO_GL
python examples/06_keyboard_control_arm.py --camera front
```

按键映射：

```text
A / D : panda_joint1 -/+
W / S : panda_joint2 +/-
Q / E : panda_joint3 -/+
R / F : panda_joint4 +/-
T / G : panda_joint5 +/-
Y / H : panda_joint6 +/-
Z / X : gripper open/close

P     : 在终端打印当前目标值
```

这个例子控制的是 7 维动作空间里的真实关节目标：


## 5. 采集 LeRobot 数据集

当前动作空间已经改成 7 维：

```text
panda_joint1
panda_joint2
panda_joint3
panda_joint4
panda_joint5
panda_joint6
gripper_closed
```

也就是 6 个 Panda 风格机械臂关节目标，加 1 个夹爪闭合量。旧的 4 维笛卡尔动作数据集不要和新版混用，需要重新采集。

数据采集时，环境来自 `env.py`，专家动作来自 `scripted_grasp.py`：

```python
from lerobot_smolvla_mujoco_demo import CubeGraspEnv, scripted_grasp_action

env = CubeGraspEnv()
obs, info = env.reset()
action = scripted_grasp_action(env, step=0)
obs, reward, terminated, truncated, info = env.step(action)
```

先做小规模测试：

```bash
MUJOCO_GL=egl cube-grasp-record \
  --episodes 5 \
  --steps 280 \
  --repo-id local/panda_6dof_7ctrl_test
```

正式采集建议 50 到 200 条：

```bash
MUJOCO_GL=egl cube-grasp-record \
  --episodes 100 \
  --steps 280 \
  --repo-id local/panda_6dof_7ctrl
```

输出位置：

```text
outputs/cube_grasp_raw/
outputs/lerobot_datasets/local/panda_6dof_7ctrl/
```

## 6. 启动 SmolVLA 训练

默认训练脚本：

```bash
DATASET_REPO_ID=local/panda_6dof_7ctrl \
DATASET_ROOT=outputs/lerobot_datasets \
OUTPUT_DIR=outputs/smolvla_panda_6dof_7ctrl \
STEPS=5000 \
BATCH_SIZE=16 \
bash scripts/train_smolvla_cube.sh
```

如果显存不足，先降低 batch size：

```bash
BATCH_SIZE=4 STEPS=5000 bash scripts/train_smolvla_cube.sh
```

训练输出目录：

```text
outputs/smolvla_panda_6dof_7ctrl/
```

## 7. 常见问题

### `cuda_available` 在工具里是 False

Codex 工具默认沙箱里可能看不到 `/dev/nvidia*`，因此会显示 `cuda_available=False`。在你自己的真实终端中激活 conda 环境后运行验证脚本，已确认可以看到 RTX 4090。

### `nvidia-smi` 报错

在普通沙箱命令里可能无法访问 GPU。请在系统终端直接运行：

```bash
nvidia-smi
```

### MuJoCo 无法渲染

先设置：

```bash
export MUJOCO_GL=egl
```

如果还是失败：

```bash
export MUJOCO_GL=osmesa
```

### 重新安装 CUDA 版 Torch

使用已写好的国内源脚本：

```bash
cd /home/mkls/xiao_run/lerobot_smolvla_mujoco_demo
conda activate /home/mkls/xiao_run/.conda-lerobot-smolvla
bash scripts/install_torch_cu118_cn.sh
```
