## 5. 采集 LeRobot 数据集

当前动作空间已经改成 7 维：

```text
panda_joint1
panda_joint2
panda_joint3
panda_joint4
panda_joint5
panda_joint6
gripper_closed
```

也就是 6 个 Panda 风格机械臂关节目标，加 1 个夹爪闭合量。旧的 4 维笛卡尔动作数据集不要和新版混用，需要重新采集。

当前训练数据格式是双相机 + 7 维状态：

```text
observation.images.front
observation.images.top_oblique
observation.state = 6 个关节角 + gripper_closed
action = 6 个关节目标 + gripper_closed
```

目标方块位置、托盘位置、`cube_on_goal` 不再放进 `observation.state`。模型需要从两路图像里判断目标方块位置。

数据采集时，环境来自 `env.py`，专家动作来自 `scripted_grasp.py`：

```python
from lerobot_smolvla_mujoco_demo import CubeGraspEnv, scripted_grasp_action

env = CubeGraspEnv()
obs, info = env.reset()
action = scripted_grasp_action(env, step=0)
obs, reward, terminated, truncated, info = env.step(action)
```

先做小规模测试：

```bash
MUJOCO_GL=egl cube-grasp-record \
  --episodes 5 \
  --steps 280 \
  --raw-dir outputs/cube_grasp_dualcam_state7_test_raw \
  --lerobot-root outputs/lerobot_datasets_dualcam_state7_test \
  --repo-id local/panda_6dof_7ctrl_dualcam_state7_test \
  --success-only
```

正式采集建议 50 到 200 条：

```bash
MUJOCO_GL=egl cube-grasp-record \
  --episodes 100 \
  --steps 280 \
  --raw-dir outputs/cube_grasp_dualcam_state7_100_raw \
  --lerobot-root outputs/lerobot_datasets_dualcam_state7 \
  --repo-id local/panda_6dof_7ctrl_dualcam_state7 \
  --success-only
```

训练 SmolVLA 时，不建议把失败的手动抓取轨迹混进去。推荐优先用脚本专家采集成功数据：

```text
脚本专家数据 80% 到 100%
键盘末端控制成功数据 0% 到 20%
失败轨迹先不要加入训练集
```

原因是 SmolVLA 做行为克隆时会学习你给它的动作。如果数据里大量包含“对不准、夹空、乱动”的轨迹，模型会把这些也学进去。

带 `--success-only` 时，只保存完整成功轨迹。当前成功标准是：

```text
本条 episode 中曾经成功抬起蓝色方块
并且最终 cube_on_goal=True，也就是蓝色方块进入托盘
```

只抬起但没有放进托盘，不会保存。

输出位置：

```text
outputs/cube_grasp_raw/
outputs/lerobot_datasets_dualcam_state7/
```

## 6. 启动 SmolVLA 训练

第一次训练前，先下载 SmolVLA 预训练权重和底层 SmolVLM 权重：

```bash
cd /home/mkls/xiao_run/lerobot_smolvla_mujoco_demo
conda activate /home/mkls/xiao_run/.conda-lerobot-smolvla
bash scripts/download_smolvla_models.sh
```

脚本默认使用：

```text
HF_ENDPOINT=https://huggingface.co
HF_HUB_DISABLE_XET=1
```

之前默认的 `https://hf-mirror.com` 在当前机器上出现过 SSL 中断。如果你的网络能稳定访问该镜像，也可以手动指定：

```bash
HF_ENDPOINT=https://hf-mirror.com bash scripts/download_smolvla_models.sh
```

使用当前已经采集好的 200 条成功轨迹训练 5 个 epoch：

```bash
BATCH_SIZE=4 \
EPOCHS=5 \
LOG_FREQ=10 \
bash scripts/train_smolvla_with_logs.sh
```

这个训练脚本会自动做两件事：

1. 使用本地缓存离线加载模型，避免训练中反复联网检查 HuggingFace 文件。
2. 生成 `outputs/pretrained/smolvla_panda_dualcam_state7_base/`，把 `lerobot/smolvla_base` 的默认配置改成当前数据集使用的 `front` + `top_oblique` 双相机、7 维状态、7 维动作。

当前数据集是 200 条 episode、51000 帧。脚本会按下面公式把 epoch 自动换算成 LeRobot 的 `steps`：

```text
steps = ceil(EPOCHS * 总帧数 / BATCH_SIZE)
```

所以默认：

```text
EPOCHS=5, BATCH_SIZE=4 -> STEPS=63750
```

如果你想手动指定训练步数，也可以直接传 `STEPS`，这时 `EPOCHS` 不再参与换算：

```bash
BATCH_SIZE=4 STEPS=100000 bash scripts/train_smolvla_with_logs.sh
```

训练日志会保存到：

```text
outputs/train_logs/
```

### 继续上次模型训练

可以继续上次 checkpoint 训练。当前已有模型在：

```text
outputs/smolvla_panda_dualcam_state7_200_success_test/checkpoints/010000/pretrained_model
```

继续训练到总共 5 个 epoch：

```bash
RESUME=1 \
BATCH_SIZE=4 \
EPOCHS=5 \
LOG_FREQ=10 \
OUTPUT_DIR=outputs/smolvla_panda_dualcam_state7_200_success_test \
bash scripts/train_smolvla_with_logs.sh
```

如果目录里没有 `checkpoints/last`，脚本会自动找已有的最新 checkpoint。你也可以手动指定：

```bash
RESUME=1 \
RESUME_CONFIG=outputs/smolvla_panda_dualcam_state7_200_success_test/checkpoints/010000/pretrained_model/train_config.json \
BATCH_SIZE=4 \
EPOCHS=5 \
bash scripts/train_smolvla_with_logs.sh
```

注意：继续训练时 `STEPS` 表示目标总步数，不是追加步数。比如上次已经训练到 `10000` 步，`EPOCHS=5` 换算成 `63750` 步后，会从 `10000` 继续跑到 `63750`。

默认训练脚本：

```bash
DATASET_REPO_ID=local/panda_6dof_7ctrl_dualcam_state7 \
DATASET_ROOT=outputs/lerobot_datasets_dualcam_state7 \
OUTPUT_DIR=outputs/smolvla_panda_dualcam_state7 \
EPOCHS=5 \
BATCH_SIZE=16 \
bash scripts/train_smolvla_cube.sh
```

如果显存不足，先降低 batch size：

```bash
BATCH_SIZE=4 EPOCHS=5 bash scripts/train_smolvla_cube.sh
```

训练输出目录示例：

```text
outputs/smolvla_panda_dualcam_state7_200_success_test/
```

这个 demo 会把两路相机、7 维关节状态送进 SmolVLA，输出 7 维动作控制仿真机械臂。

如果观察到模型已经把 cube 带到托盘上方，但夹爪没有稳定释放，或者搬运过程中夹爪输出抖动导致 cube 掉落，建议测试时加：

```bash
--gripper-latch
```

这个参数只影响测试/部署时的夹爪控制逻辑，不会改变训练数据和模型权重。逻辑是：模型在 cube 附近输出闭合意图后，程序保持夹爪闭合；到托盘附近且模型输出低于释放阈值时，再松开夹爪。

```bash
  --episodes 5 \
  --steps 280 \
  --gripper-latch \
  --out outputs/policy_eval_5epoch_latch_5eps.mp4
```

## 8. 常见问题

### `cuda_available` 在工具里是 False

Codex 工具默认沙箱里可能看不到 `/dev/nvidia*`，因此会显示 `cuda_available=False`。在你自己的真实终端中激活 conda 环境后运行验证脚本，已确认可以看到 RTX 4090。

### `nvidia-smi` 报错

在普通沙箱命令里可能无法访问 GPU。请在系统终端直接运行：

```bash
nvidia-smi
```

### MuJoCo 无法渲染

先设置：

```bash
export MUJOCO_GL=egl
```

如果还是失败：

```bash
export MUJOCO_GL=osmesa
```

### 重新安装 CUDA 版 Torch

使用已写好的国内源脚本：

```bash
cd /home/mkls/xiao_run/lerobot_smolvla_mujoco_demo
conda activate /home/mkls/xiao_run/.conda-lerobot-smolvla
bash scripts/install_torch_cu118_cn.sh
```
